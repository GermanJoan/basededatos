Definiciones de Base de Datos

1. ¿Qué es un Dato?

Un dato es una representación simbólica de una característica o atributo de una entidad. Los datos pueden ser números, letras, o cualquier otro símbolo que describa una propiedad específica de un objeto o evento.

2. ¿Qué es información?

La información es un conjunto de datos procesados y organizados de manera que tienen significado y relevancia. La información permite la toma de decisiones y la comprensión de eventos o situaciones.

3. ¿Diferencia entre dato e información?

La principal diferencia entre dato e información es que los datos son hechos crudos y sin procesar, mientras que la información es el resultado de procesar y organizar esos datos de manera que sean útiles y significativos.

4. ¿Tipos de datos?

Los tipos de datos pueden clasificarse en varios tipos, incluyendo:

  • Datos numéricos: enteros y decimales.
  • Datos de texto: caracteres, cadenas de texto.
  • Datos booleanos: verdadero o falso.
  • Datos de fecha y hora: fechas y tiempos específicos.
  • Datos binarios: imágenes, vídeos, sonidos.

5. ¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear un modelo para la organización y estructura de los datos. Este modelo define cómo se almacenan, organizan y relacionan los datos en un sistema de información.

6. ¿Tipos de bases de Datos, según:

Variabilidad:

  • Bases de datos estáticas: no permiten modificaciones una vez creadas.
  • Bases de datos dinámicas: permiten la inserción, modificación y eliminación de datos.

Contenido:

  • Bases de datos bibliográficas.
  • Bases de datos de texto completo.
  • Bases de datos numéricas.

Modelo de base de datos:

  • Bases de datos jerárquicas.
  • Bases de datos de red.
  • Bases de datos relacionales.
  • Bases de datos orientadas a objetos.
  • Bases de datos NoSQL.

7. ¿Qué es el modelo relacional?

El modelo relacional es un tipo de modelo de base de datos en el cual los datos se organizan en tablas bidimensionales llamadas relaciones. Cada tabla está compuesta por filas y columnas, y cada fila representa un registro único con un conjunto de atributos.

8. ¿Describa los elementos del Diagrama entidad Relacional?

Los elementos del Diagrama Entidad-Relación (ERD) incluyen:

  • Entidades: Representan objetos o conceptos del mundo real que tienen existencia independiente.
  • Atributos: Son las propiedades o características de una entidad.
  • Relaciones: Describen cómo las entidades se relacionan entre sí.
  • Llaves primarias: Identificadores únicos de una entidad.
  • Llaves foráneas: Campos que establecen la relación entre entidades.

9. ¿Qué es una entidad?

Una entidad es cualquier objeto, persona, lugar o evento del mundo real que se puede identificar de manera única y que tiene existencia independiente en una base de datos.

10. ¿Qué es un atributo y sus tipos?

Un atributo es una propiedad o característica de una entidad. Los tipos de atributos incluyen:

  • Atributo simple: No puede dividirse en más partes (ejemplo: edad).
  • Atributo compuesto: Puede dividirse en subpartes más pequeñas (ejemplo: nombre completo puede dividirse en nombre y apellido).
  • Atributo derivado: Su valor se puede calcular a partir de otros atributos (ejemplo: edad calculada a partir de la fecha de nacimiento).
  • Atributo multivaluado: Puede tener múltiples valores para una sola entidad (ejemplo: número de teléfono).

11. ¿Qué es una relación?

Una relación es una asociación entre dos o más entidades. Describe cómo las entidades se conectan y se interactúan entre sí en una base de datos.

12. ¿Qué son las llaves primarias, foránea y sus características?

Llave primaria: Es un atributo o conjunto de atributos que identifica de manera única a cada registro en una tabla. Debe ser único y no nulo.

Llave foránea: Es un atributo o conjunto de atributos en una tabla que establece una relación con la llave primaria de otra tabla. Permite mantener la integridad referencial entre tablas.

13. ¿Qué son las reglas del Negocio?

Las reglas del negocio son las políticas, procedimientos y restricciones que determinan cómo se deben gestionar y operar los datos en una organización. Estas reglas aseguran que los datos sean manejados de manera consistente y correcta.

14. ¿Qué es la cardinalidad o mapeo?

La cardinalidad o mapeo describe el número de instancias de una entidad que pueden asociarse con una instancia de otra entidad en una relación. Los tipos de cardinalidad incluyen:

  • Uno a uno: Una instancia de una entidad se relaciona con una única instancia de otra entidad.
  • Uno a muchos: Una instancia de una entidad se relaciona con múltiples instancias de otra entidad.
  • Muchos a muchos: Múltiples instancias de una entidad se relacionan con múltiples instancias de otra entidad.
